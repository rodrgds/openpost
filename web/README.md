# OpenPost Web

SvelteKit frontend for [OpenPost](../README.md) — a lightweight, self-hosted social media scheduler.

## Tech Stack

- **SvelteKit 5** with runes (`$state`, `$derived`, `$effect`)
- **TailwindCSS 4** for styling
- **shadcn-svelte** UI component library
- **Paraglide** for i18n (English, Spanish, Portuguese)
- **openapi-fetch** for typed API client (generated from backend OpenAPI spec)
- **Vitest** for unit testing
- **Capacitor** for Android mobile app support

## Project Structure

```
web/
├── src/
│   ├── lib/
│   │   ├── api/           # Typed API client + generated types
│   │   ├── components/    # UI components (app + shadcn primitives)
│   │   ├── stores/        # Svelte stores (auth, UI, Mastodon instance)
│   │   ├── paraglide/     # Auto-generated i18n runtime
│   │   └── types/         # Shared TypeScript types
│   └── routes/
│       ├── +page.svelte   # Dashboard/home
│       ├── accounts/      # Connected accounts management
│       ├── connect/       # Connect social platforms
│       ├── login/         # Login page
│       └── register/      # Registration page
├── messages/              # Paraglide i18n source files (en, es, pt)
├── openapi.json           # OpenAPI spec from backend (used for type generation)
└── capacitor.config.ts    # Capacitor config for Android build
```

## Routes

| Route | Description |
|-------|-------------|
| `/` | Dashboard — workspace overview, recent posts, schedule summary |
| `/accounts` | Manage connected social accounts (X, Mastodon, Bluesky, LinkedIn, Threads) |
| `/accounts/mastodon/callback` | Mastodon OAuth callback handler |
| `/connect` | Connect new social platform accounts |
| `/login` | User login |
| `/register` | User registration |

## API Client

The typed API client is generated from the backend's OpenAPI spec:

1. Backend exposes `/openapi.json` (generated by Huma)
2. Copy to `web/openapi.json`
3. Run `bun run generate:types` to regenerate `src/lib/api/types.d.ts`
4. Client in `src/lib/api/client.ts` uses `openapi-fetch` for type-safe API calls

## i18n

Translations are managed with Paraglide. Source messages are in `messages/*.json`.
Supported languages: English (`en`), Spanish (`es`), Portuguese (`pt`).

To add translations:
1. Edit the relevant `messages/{lang}.json` file
2. Run `bunx @inlang/cli machine translate` (or edit manually)
3. The Paraglide runtime is auto-generated in `src/lib/paraglide/`

## Android (Capacitor)

OpenPost can be built as an Android app using Capacitor:

```bash
bun run build:capacitor
cd android
./gradlew assembleDebug
```

## Development

The frontend can run independently against a local backend:

```bash
bun run dev
```

The dev server runs on `http://localhost:5173` and proxies API requests to the backend at `http://localhost:8080`.
